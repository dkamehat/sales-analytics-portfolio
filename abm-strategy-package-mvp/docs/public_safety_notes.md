# Public Safety Notes

This portfolio project is intentionally fictional and generic.

It is meant to demonstrate operating design, not disclose real company practices.

## Allowed content

The following content is appropriate for this public project:

- generic ABM strategy concepts
- fictional account names
- fictional stakeholder roles
- fictional engagement summaries
- generic tiering criteria
- synthetic CSV data
- reusable templates
- prompt examples for synthetic or approved data

## Do not add

Do not add any of the following:

- real customer names
- real company names from non-public work
- real revenue, margin, or KPI values
- proprietary SQL
- screenshots of internal tools
- CRM/SFA exports
- private meeting notes
- internal organization names
- personal data
- confidential strategy, pricing, or contract information
- anecdotes that can identify a specific employer, customer, team, or meeting

## Abstraction rules

### Use fictional accounts

Good:

```text
Northstar Manufacturing Co.
Blue River Logistics
Metro Retail Holdings
```

Bad:

```text
Real customer or employer names
```

### Use ranges or labels instead of numbers

Good:

```text
Revenue Band: high
Risk: medium
Priority Score: 4.2 based on fictional sample data
```

Bad:

```text
Actual revenue, order count, margin, contract value, or KPI values
```

### Use generic systems

Good:

```text
CRM, BI, ERP, Data Warehouse
```

Bad:

```text
Specific internal system names or unreleased architecture
```

### Keep organization descriptions generic

Good:

```text
coordination-heavy organization
stakeholder-heavy decision process
cross-functional review
```

Bad:

```text
specific number of leaders, specific meeting cadence, specific internal team structure
```

## External LLM usage

Use only synthetic, public, or formally approved data with external LLMs.

Do not paste confidential data or personal data into prompts.

## Purpose of this project

This project is a portfolio artifact for showing how to:

```text
name business objects
convert decisions into functions
attach actions to decisions
record review and audit logs
```

It is not a record of any specific company's operations.
